# Phase 07 集成验收、联调与优化

## 1. 阶段目标

完成三维态势展示应用的端到端验收、真实服务联调、构建验证、性能检查和上线前风险收敛。此阶段不再引入大规模功能，重点是证明系统稳定可用。

## 2. 实现思路

先补齐自动化测试和 e2e mock 测试，再使用真实三维瓦片地址和真实拓扑接口联调。联调过程中重点验证 CORS、Cesium 静态资源路径、轮询稳定性、无效坐标处理、summary 数量校验、实体增量更新和长时间运行性能。发现问题后优先回到对应阶段修正，而不是在验收阶段堆临时补丁。

## 3. 需要新增的文件

- `e2e/situation.spec.ts`
- `src/test/mocks/cesium.ts`，如前面阶段尚未统一 Cesium mock。
- `docs/deploy-notes.md`，如需要记录部署和代理说明。
- `docs/verification-report.md`，如需要沉淀验收报告。

## 4. 需要修改的文件

- `playwright.config.ts`，仅当需要配置接口 mock、测试超时或 WebGL 浏览器参数时修改。
- `README.md`，如需要补充运行、配置和验证命令。
- 前面阶段遗留的测试或实现文件。

## 6. 验收标准

- 全量类型检查、单元测试、构建和 e2e 测试通过。
- 真实三维瓦片可以加载，无 Cesium Worker、Assets、Widgets 404。
- 真实拓扑接口可以轮询，失败时保留旧数据并显示错误状态。
- 根级 `summary.node_count` 和 `summary.link_count` 校验逻辑正确。
- 节点、轨迹、链路、图例、状态栏和标牌交互满足 `001-final.md` 的验收标准。
- 长时间运行无明显实体泄漏、定时器泄漏或浏览器崩溃。
- 部署配置和验证命令已记录。

## 7. 风险与注意事项

- 真实局域网接口可能受网络、CORS 或服务状态影响，验收报告需要区分前端缺陷和外部环境问题。
- 若节点数量远超第一版预估，性能优化应作为独立迭代，不建议在验收阶段临时重写渲染架构。
