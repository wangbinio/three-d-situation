# 新增无人机类型

在原有 @plan/plan/001-总体设计/001-final.md 总体设计中定义了：

```5.4 节点类型映射
节点类型枚举固定为 8 类。1-8。
```

这8类是通过解析json中节点的`node_type`获取到的。现在需要额外增加1个特殊类型：无人机，类型id设为30（可配置）。

但是这个类型并不是通过`node_type`获取，而且通过json中节点的`node_manage_ip_addr`属性中的ip地址来判断。

ip处于这个范围内，那么就忽略`node_type`中的类型，判定是无人机，ip也要做成可以配置的。
- 172.16.6.11-172.16.6.18
- 172.16.7.20-172.16.7.30
- 172.16.8.32-172.16.8.42
- 172.16.9.44-172.16.9.73

无人机类型图标使用`public/models/node-icons/svg/无人机.svg`


## 无人机类型特殊处理

当节点目标的类型是无人机，在从后端topology接口获取数据并解析之后，准备加入到目标历史航迹列表之前，要额外处理：

  1. 判断位置信息中的高度是否大于1，如果大于1，不做任何特殊处理, 直接将数据加入历史轨迹列表即可。
  2. 判断历史轨迹列表size是否大于1，如果小于1, 不做任何特殊处理, 直接将数据加入历史轨迹列表即可。
  3. 判断列表中最后一个位置和要加入的这个位置，距离是否大于10m，如果小于10, 不做任何特殊处理, 直接将数据加入历史轨迹列表即可。
  4. 如果距离大于10m，那么将要加入的这个位置的高度改为100m（也是可配置参数）。
  5. 同时也用这个最新带高度的位置信息来显示无人机的位置，而不是直接解析json得到的高度0。

因为有些无人机没有正确的高度值，都是0。这个处理的逻辑就是通过位置变化的大小来判断无人机是否在运动状态，如果在运动，那么就给默认的高度100。
